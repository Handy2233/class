# GEC6818 智能触摸控制台

基于 GEC6818 / S5P6818 开发板的综合交互项目。

项目集成 LCD 图形显示、触摸输入、电子相册、环境数据监测、烟雾报警、LED 控制、蜂鸣器控制、配置持久化和历史报警记录。

---

## 项目背景

嵌入式 Linux 开发不仅需要驱动外设，还需要把数据显示、触摸交互、设备控制和异常报警组织成完整应用。

本项目以 GEC6818 开发板为平台，完成一个面向课程实验和演示的智能触摸控制台：

- 通过 LCD 提供图形化页面。
- 通过触摸屏完成页面跳转和设备控制。
- 通过串口读取 GY-39 环境数据和 Z-MQ-01 烟雾浓度。
- 通过蜂鸣器和历史记录完成报警闭环。

---

## 硬件与外设

| 模块 | 作用 | 默认接口 |
| --- | --- | --- |
| LCD 屏 | 页面显示、图片显示、文字渲染 | `/dev/fb0` |
| 触摸屏 | 点击、滑动、页面切换 | `/dev/input/event0` |
| GY-39 | 光照、温度、湿度等环境数据 | `/dev/ttySAC1` |
| Z-MQ-01 | 烟雾浓度检测 | `/dev/ttySAC2` |
| LED D7-D10 | 触摸控制输出 | `/dev/led_drv` 或 sysfs |
| 蜂鸣器 | 手动控制和报警联动 | `/dev/pwm` |

## 核心功能

项目主界面提供三个入口：

- 电子相册：浏览 `pic/` 目录下的图片，支持左右滑动切换。
- 数据监测：显示 GY-39 和 Z-MQ-01 的实时数据，并根据阈值判断报警。
- LED 灯及蜂鸣器控制：触摸控制 D7-D10 LED 和蜂鸣器。

数据监测页还可以进入历史报警页，查看报警次数、报警时间和事件详情。

## 页面结构

```text
首页 SMART PANEL
├── 电子相册
│   └── 左右滑动切换图片
├── 数据监测
│   ├── 温度、湿度、光照、烟雾浓度
│   ├── 温度阈值、烟雾阈值
│   └── 历史报警
└── LED 灯及蜂鸣器控制
    ├── LED D7-D10
    └── 蜂鸣器
```

页面定义集中在 `lib/inc/ui_app_data.h`，动态绘制和触摸处理集中在 `lib/src/ui_app.c`。

## 报警功能

报警判断基于两个阈值：

- 温度阈值：默认 `30 C`。
- 烟雾阈值：默认原始值 `300`。

当温度或烟雾浓度超过阈值时：

1. 数据监测页显示异常状态。
2. 历史报警次数增加。
3. 报警事件写入日志文件。
4. 若声音报警开关开启，则蜂鸣器响起。

---

## 历史报警

历史报警记录包含：

- 报警时间。
- 报警事件，例如温度报警、烟雾报警、温度/烟雾报警。
- 报警时的传感器值和阈值。

日志文件默认路径：

```text
./alarm_history.log
/root/alarm_history.log
/tmp/alarm_history.log
```

程序会尽量从可用路径恢复历史报警次数和最近报警事件。

## 配置持久化

配置文件默认路径：

```text
./ui_app.conf
```

当前配置项：

```ini
temperature_alarm_c=30
smoke_alarm_raw=300
alarm_sound=1
```

这些配置用于保存温度阈值、烟雾阈值和声音报警开关，避免程序重新运行后设置丢失。

## 非阻塞事件循环

`main.c` 只保留应用入口：

```c
int main(void)
{
    if(ui_app_init() != 0)
        return 1;

    while(1) {
        if(ui_app_update() != 0)
            break;
    }

    ui_app_cleanup();
    return 0;
}
```

触摸、页面刷新和传感器读取由 `ui_app_update()` 统一驱动。

---

## 非阻塞传感器读取

传感器读取采用非阻塞 UART 和状态机：

- 串口以 `O_NONBLOCK` 打开。
- 主循环中每次只推进一个小步骤。
- GY-39 和 Z-MQ-01 的接收数据先进入缓冲区。
- 收到完整帧后再解析并更新页面。

这种设计避免传感器等待过程阻塞触摸响应。

## 模块分工

| 模块 | 文件 | 职责 |

| LCD | `lcd.c` / `lcd.h` | framebuffer 初始化、绘图、图片缩放 |

| LED | `led.c` / `led.h` | D7-D10 控制 |
| PWM | `pwm.c` / `pwm.h` | 蜂鸣器控制 |
| UI 应用 | `ui_app.c` / `ui_app.h` | 页面、报警、配置、主循环 |

## 字体显示方案

项目使用字体文件直接渲染 UTF-8 文本：

- 中文字体：`font/SimSun.ttc`
- 西文字体：`font/Consolas.ttf`
- 字体配置：`font/word.json`

`word.c` 负责：

- 解码 UTF-8 字符。
- 区分中文和西文。
- 使用 `stb_truetype` 栅格化字形。
- 将文字混合绘制到 framebuffer。

## 构建方式

本项目使用 CMake 和 ARM 交叉编译工具链。

配置：

```bash
cmake -S . -B build -DCMAKE_TOOLCHAIN_FILE=arm-linux-toolchain.cmake
```

编译：

```bash
cmake --build build
```

生成程序：

```text
build/main
```

---

## 部署运行

将程序和资源文件放到开发板同一目录，例如 `/project`：

```text
/project
├── main
├── ui_app.conf
├── font/
└── pic/
```

运行：

```bash
cd /project
./main
```

如果配置文件需要在程序中修改，运行用户必须对 `ui_app.conf` 所在目录和文件有写权限。

## 演示流程

建议课堂或答辩演示按以下顺序进行：

1. 展示首页，说明三个功能入口。
2. 进入电子相册，滑动切换图片。
3. 进入数据监测，展示温度、湿度、光照和烟雾数据。
4. 调整或触发阈值，演示报警和蜂鸣器联动。
5. 打开历史报警，展示报警时间和事件记录。
6. 进入设备控制页，触摸控制 LED 和蜂鸣器。
7. 退出并重新运行，说明配置和历史记录可恢复。

## 项目特点

- 模块边界清晰：入口、UI、外设、协议和字体渲染分层实现。
- 用户体验完整：具备首页、相册、监测、控制、报警历史多个页面。
- 数据闭环完整：采集、判断、报警、记录、恢复全部覆盖。
- 响应方式合理：通过非阻塞串口和状态机减少触摸延迟。
- 便于扩展：新增传感器、页面或设备时可以在对应模块内扩展。

# 2-传感器模块

GY39、MQ2,都是串口模块，基于串口通信



## 1.UART串口通信基本原理

串口是一种异步的串行通信方式。

-   串行：一根数据线，数据1bit位1bit位传输
-   并行：多跟数据线同时传输

**需要利用串口通信，需要以下数据线：**

-   Tx: tansport，发送数据线，发送数据
-   Rx: recv, 接收数据线，接收数据

```
eg: 设备A 与 设备B 利用串口通信
	A		B
	Tx ---- Rx
	Rx ---- Tx
	VCC --  VCC
	GND --  GND
```



## 2.串口通信的数据帧格式

串口数据以帧位单位

**一帧完整串口数据 = 起始位 + 数据位 + 校验位 + 停止位**

-   起始位：（固定1位）

    告诉接收方，数据来了，请准备接收

    为1个周期的低电平信号，Tx发送数据线在空闲状态时保持为高电平

-   数据位：（5~8bit, 最常用8bit）

    真正要传输的有效数据，遵循低位先发，高位后发

    嵌入式领域一般都是发送8bit

-   校验位：三种模式

    -   无校验（N）：没有校验位，最常用，速度最快
    -   奇校验：8位数据位 + 校验位中1的总数为奇数个
    -   偶校验：8位数据位 + 校验位中1的总数为偶数个

-   停止位：（1 / 2， 最常用1位）
    -   高电平信号，表示一帧数据传输结束

**串口通信时，双方需要约定数据帧格式，除开数据帧格式外，还需要约定通信的波特率，波特率决定了传输1bit所需要的时间。**

波特率: bps（bit per second） 每秒钟可以传输的bit数，决定数据传输的速率，常见的波特率：9600bps   38400bps   115200bps ........



## 3.Linux下串口初始化

```C
#include <stdio.h>
#include <fcntl.h> 
#include <unistd.h>
#include <termios.h> 
#include <stdlib.h>
#include <netdb.h>
#include <string.h>
#include <errno.h>

#define COM2 "/dev/ttySAC1"  //串口文件
#define COM3 "/dev/ttySAC2"  //串口文件
#define COM4 "/dev/ttySAC3"  //串口文件

/* 设置串口参数（初始化串口） */
void init_tty(int fd)
{    
	//声明设置串口的结构体
	struct termios termios_new;
	//先清空该结构体
	bzero( &termios_new, sizeof(termios_new));
	//	cfmakeraw()设置终端属性，就是设置termios结构中的各个参数。
	cfmakeraw(&termios_new);
	//设置波特率
	//termios_new.c_cflag=(B9600);
	cfsetispeed(&termios_new, B9600);
	cfsetospeed(&termios_new, B9600);
	//CLOCAL和CREAD分别用于本地连接和接受使能，因此，首先要通过位掩码的方式激活这两个选项。    
	termios_new.c_cflag |= CLOCAL | CREAD;
	//通过掩码设置数据位为8位
	termios_new.c_cflag &= ~CSIZE;
	termios_new.c_cflag |= CS8; 
	//设置无奇偶校验
	termios_new.c_cflag &= ~PARENB;
	//一位停止位
	termios_new.c_cflag &= ~CSTOPB;
	tcflush(fd,TCIFLUSH);
	// 可设置接收字符和等待时间，无特殊要求可以将其设置为0
	termios_new.c_cc[VTIME] = 10;
	termios_new.c_cc[VMIN] = 1;
	// 用于清空输入/输出缓冲区
    tcflush (fd, TCIFLUSH);
	//完成配置后，可以使用以下函数激活串口设置
	if(tcsetattr(fd,TCSANOW,&termios_new) )
		printf("Setting the serial1 failed!\n");
}
```



## 4.GY39温湿度及光照强度检测模块

​	GY-39 是一款低成本，气压，温湿度，光强度传感器模块，工作电压 3-5v，功耗小，安装方便。



### 4.1接线

任选一组开发板上的串口（COM2  / COM3  / COM4）

注意：在给开发板上电之前，一定要检查线连接正确的，保证没有反接

```
开发板         GY39模块
GND   ------  GND
TX    ------  DR 
RX    ------  CT
5V    ------  VCC
```



### 4.2模块命令字节

**外部控制器发送至 GY-39 模块**，若想要获取GY39模块的数据，必须先给模块发送指令，触发模块开始工作，获取环境中的温湿度、海拔、光照等数据。

**指令格式：帧头+指令+校验和(8bit) **

-   帧头：0xa5 
-   指令：光照强度指令 / 温湿度指令
    -   光照强度：0x81
    -   温湿度：0x82

-   校验和：帧头 + 指令的和，取其低8位
    -   光照强度：0x26
    -   温湿度：0x27

**完成指令格式：**

-   **获取光照强度指令：** 0xa5   0x81  0x26
-   **获取温湿度指令：** 0xa5   0x82   0x27



### 4.3模块回送数据

**光照强度数据：** 0x5A(帧头) + 0x5A(帧头) + 0x15(表明回送的是光强数据) + 0x04(光照强度数值数据量) + 4字节光照强度数据 + 1字节校验和

-   光照强度返回的完整的数据一共有 9 字节，其中byte4 ~ byte7就是要获取光强数据

-   光强数据处理： lux = byte4 << 24 | byte5 << 16 | byte6 << 8 | byte7;

    ​                            lux /= 100;



**温湿度数据：** 0x5A(帧头) + 0x5A(帧头) + 0x45(表明回送的数据是温湿度数据) + 0x0A(温湿度大气压数据量) + 2字节温度数据 + 4字节气压数据 + 2字节湿度数据 + 2字节海拔数据 + 1字节校验和

-   温湿度、气压、海拔完整的数据一共回送 15 个字节，其中Byte4~Byte5 为温度数据，其中Byte6~Byte9为气压数据，其中Byte10~Byte11湿度数据，其中Byte12~Byte13为海拔数据

-   温度数据处理:  T = byte4 << 8 | byte5;

    ​                           T /= 100;

-   湿度数据处理：Hum = byte10 << 8 | byte11;

    ​                           Hum /= 100;



## 5.编程实现获取模块数据

```C
#include <unistd.h>
#include <termios.h>

int lux = 0;

/* 
    gy39_get_data: gy39模块获取数据
    @COMx: 指定连接的串口,起始就是连接的串口的文件路径
    返回值:
        成功 0
        失败 -1

    eg: gy29_get_data(COM3);
*/
int gy39_get_data(char *COMx)
{
    //1.打开连接的串口文件
    int uart_fd = open(COMx, O_RDWR);
    if(uart_fd == -1)
    {
        printf("open %s fail\n", COMx);
        return -1;
    }

    //2.初始化串口（串口通信之前，双方必须要约定串口数据帧格式 + 串口通信的波特率）
    init_tty(uart_fd);

    //3.控制器（开发板）给GY39模块发送获取光照强度指令
    char lux_cmd[3] = {0xa5, 0x81, 0x26};
    write(uart_fd, lux_cmd, 3);
    sleep(1); //等待模块响应并回送数据

    //4.获取GY39模块回送的光照强度数据
    char lux_data[9] = {0};
    read(uart_fd, lux_data, 9);
    if(lux_data[0] == 0x5A && lux_data[1] == 0x5A && lux_data[2] == 0x15)
    {
        lux = lux_data[4] << 24 | lux_data[5] << 16 | lux_data[6] << 8 | lux_data[7];
        lux /= 100;
    }

    //5.控制器（开发板）给GY39模块发送获取温湿度指令
    char t_hum_cmd[3] = {0xa5, 0x82, 0x27};
    write(uart_fd, t_hum_cmd, 3);
    sleep(1); //等待其响应成功

    //6.获取GY39模块回送的温湿度数据
    char t_hum_data[15] = {0};
    read(uart_fd, t_hum_data, 15);
    if(t_hum_data[0] == 0x5A && t_hum_data[1] == 0x5A && t_hum_data[2] == 0x45)
    {
        t = t_hum_data[4] << 8 | t_hum_data[5];
        t /= 100;

        hum = t_hum_data[10] << 8 | t_hum_data[11];
        hum /= 100;
    }

    //7.关闭串口通信文件
    close(uart_fd);
}
```
